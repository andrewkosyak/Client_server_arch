HTTP или HyperText Transfer Protocol – это протокол передачи гипертекста.

Основные характеристики HTTP протокола:
- Работает по технологии "клиент-сервер".
- Относится к седьмому уровню модели OSI.
- Относится к семейству протоколов TCP/IP( Transmission Control Protocol/Internet protocol).
- Для идентификации ресурса использует использует URI.
- Не имеет промежуточных состояний между запросом и ответом.
- Синхронный, но позволяет клиенту отправить несколько запросов подряд, не дожидаясь ответа 
сервера, при условии, что сервер даст ответы на запросы в том порядке. как они приходили.

Самым распространенным примером клиента HTTP протокола является браузер, вот самые популярные клиенты HTTP протокола:

    Google Chrome;
    Mozilla FireFox;
    Opera;
    Internet Explorer;
    Яндекс Браузер;
    Safari.
Серверы HTTP протокола можно разделить на два вида: исходные сервера и прокси-сервера.
Исходные - те, к которым обращается клиент.
Прокси - транззитные сервера меду клиентом и тем сервером, к которому он желает обратиться.

HTTP протокол имеет требования к приложениям, которые используют его для передачи данных. Требований в HTTP очень много, 
но требования можно разделить на три категории: необходимые требования,  это такие требования, которые должны выполняться
 всеми программами; рекомендуемые 
требования, это те требования, которые следует выполнять приложениям и возможные требования, которые могут не выполняться приложениями.

Структура HTTP:
1. Стартовая строка (строка состояния), которая определяет тип HTTP сообщения. Обязательна.
2. Заголовок HTTP сообщения, может включать поле Host или несколько полей для передачи служебной информации.
3. Тело HTTP сообщения, которое содержит HTTP объекты. 

Параметры HTTP протокола:
1) Версия. Обязательный параметр в первой строке любого HTTP сообщения.
2) URI - позволяет идентифицировать ресурс, которй хочет запросить клиент.
3) Дата и время.
4) Кодирование сообщений и кодирование передачи.
5) Тип данных или медиа тип
6) HTTP протокол имеет параметры, которые позволяют определить язык сообщения, в HTTP это называется метка языка.

Байт - основная единица измерения в HTTP протоколе.
HTTP сообщение - основной способ общения в HTTP протоколе.
HTTP запрос - это сообщение, которое клиент отправляет серверу.
HTTP ответ - это сообщение, которое сервер посылает клиенту.


HTTPS (от англ. HyperText Transfer Protocol Secure — безопасный протокол передачи гипертекста) — это расширение протокола HTTP,
 поддерживающее шифрование посредством криптографических протоколов SSL и TLS.

- HTTPS - это не отдельный протокол передачи данных, а  расширение протокола HTTP с надстройкой шифрования;
- HTTPS обеспечивает конфиденциальность информации путем ее шифрования;
- HTTP использует порт 80, HTTPS — порт 443.

Обеспечение безопасной передачи данных необходимо на сайтах, где вводится и передается конфиденциальная информация (личные данные пользователей,
 детали доступа, реквизиты платежных карт) — на любых сайтах с авторизацией, взаимодействием с платежными системами, почтовыми сервисами.
 Шифрование таких данных позволит предотвратить их получение и использование третьими лицами.
