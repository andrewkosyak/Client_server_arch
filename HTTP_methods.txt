Методы в HTTP протоколе определяет способ обращения клиента к серверу и, соответственно, какую информацию хочет
 получить или передать клиент серверу
Методы в HTTP протоколе делятся на безопасные и идемпотентные. 
Безопасные - GET, HEAD. 
Идемпотентные -  GET, HEAD, PUT, DELETE, OPTIONS и TRACE.

GET - используется для получения информации от сервера по заданному URI. Запросы клиентов должны получать только 
данные и не должны никак влиять на эти данные.

HEAD - работает точно так же, как GET, но в ответ сервер посылает только заголовки и статусную 
строку без тела HTTP сообщения.

POST - для отправки данных на сервер, например, из HTML форм, которые заполняет посетитель сайта. 

PUT - для загрузки содержимого запроса на указанный в этом же запросе URI.

DELETE - удаляет указанный в URI ресурс.

CONNECT - преобразует существующее соединение в тоннель.

OPTIONS - для получения параметров текущего HTTP соединения.

TRACE - создает петлю, благодаря которой клиент может увидеть, что происходит с сообщением на всех узлах передачи.
